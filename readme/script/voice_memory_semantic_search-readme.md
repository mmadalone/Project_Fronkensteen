# Voice -- Memory Semantic Search (tool script)

Semantic (meaning-based) memory search via sqlite-vec. Wraps `pyscript.memory_semantic_search` to perform pure KNN vector similarity search against embedded L2 memories. Use when regular FTS5 search returns no results, or when the user describes something in different words than how it was stored. Each query costs one embedding API call, so regular keyword search should be tried first.

## How It Works

```
Start (query, limit)
  |
  v
+---------------------------+
| Clamp limit               |
| min(max(1, limit),        |
|     max_limit)            |
+---------------------------+
  |
  v
+---------------------------+
| pyscript.memory           |
| _semantic_search          |
| (query, clamped_limit)    |
+---------------------------+
  |
  v
+---------------------------+
| Return result             |
| (response_variable)       |
+---------------------------+
```

## Features

- Semantic similarity search against L2 memory embeddings
- Configurable default and maximum result limits
- Limit clamping prevents LLM from requesting excessive results
- `continue_on_error` on the pyscript call for resilience
- Returns results via `response_variable` for LLM consumption

## Prerequisites

- Home Assistant
- Pyscript integration installed and configured
- `pyscript/memory.py` deployed with sqlite-vec support (I-2)
- `vec0.so` compiled and deployed at `/config/vec0.so`
- Memories must have embeddings (generated by `embedding_batch` nightly job)

## Installation

1. Copy `voice_memory_semantic_search.yaml` to `config/blueprints/script/madalone/`
2. Create script: **Settings -> Automations & Scenes -> Scripts -> Add -> Use Blueprint**
3. Expose the script to Assist (Settings -> Entities -> toggle)
4. Add tool description to each conversation agent's system prompt

## Configuration

<details><summary>① Core configuration</summary>

| Input | Default | Description |
|-------|---------|-------------|
| `default_limit` | `5` | Number of results when the LLM does not specify a limit |
| `max_limit` | `10` | Hard cap on results per query (LLM limit clamped to this) |

</details>

### Script Fields (passed at call time)

| Field | Required | Default | Description |
|-------|----------|---------|-------------|
| `query` | Yes | -- | Natural language query to find semantically similar memories |
| `limit` | No | `5` | Maximum number of results (1-20, clamped to max_limit) |

## Technical Notes

- **Mode:** `single` / `max_exceeded: silent`
- The limit is clamped at runtime: `max(1, min(requested_limit, max_limit))`
- Each semantic search costs one embedding API call -- instruct agents to try regular FTS5 search first
- Results are returned as a response variable so the calling agent can read and present them
